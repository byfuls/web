3. 폼 관련 태그들

폼 만들기

사용자가 웹 사이트로 정보를 보낼 수 있는 요소들은 모두 폼(form)

텍스트 상자나 버튼 같은 폼의 형태를 만드는 것은 HTML 태그를 이용하고,

그 폼에 입력한 사용자 정보를 처리하는 것은 서버 프로그래밍을 이용. (정보 저장, 검색, 수정은 서버의 데이터베이스 기반)

<form> - 폼 만들기

<input> - 사용자가 내용을 입력하는 필드를 삽입할 때나 버튼을 삽입할 때 사용

autocomplete 속성 - 자동 완성 기능

사용자가 입력했던 내용을 기억했다가 비슷한 내용을 입력할 경우, 이전에 입력했던 내용을 힌트로 보여줌 (기본값 "on")

<label> - 폼 요소에 레이블 붙이기

입력 창 옆에 붙여 놓은 텍스트

사용 방법 1. <label> 태그 안에 폼 요소를 넣는 것

사용 방법 2. <label> 요소와 폼 요소를 따로 사용하고 <label> 태그안에 for 속성을 이용하고 폼 요소에서는 id속성을 이용해 연결

라디오 버튼과 체크박스에서 사용하는 <label> 태그

label로 라디오 버튼과 체크박스를 만들면, 텍스트를 클릭해도 해당 버튼이 선택됨

<fieldset>, <legend> 태그 - 폼 요소 그룹으로 묶기

하나의 폼 안에서 여러 구역을 나누어 표시하려고 할 때 사용

<fieldset> 과 <fieldset> 태그 사이의 폼들을 하나의 영역으로 묶고 외곽선을 그려줌

<legend>태그는 <fieldset>태그로 묶은 그룹에 제목을 붙여줌

type="submit", type="reset" - 서버 전송, 리셋 버튼 넣기

reset 버튼은 요소에 입력된 모든 정보를 재설정해, 입력한 내용을 모두 지울 수 있음

submit 버튼은 사용자가 폼에 입력한 정보를 서버로 전송하는 submit 버튼을 넣음.
submit 버튼으로 전송된 정보는 <form>태그에서 지정한 폼 처리 프로그램에 넘겨짐

type="image" - 이미지 버튼 넣기

submit 버튼 대신 전송 이미지를 넣을 수 있음

type="button" - 버튼 넣기

submit, reset과 같은 기능은 없고, 오직 버튼만 넣기 때문에 스크립트 함수 등을 연결해 사용, value 속성을 사용해 버튼에 표시할 내용 지정

type="file" - 파일 첨부하기

<input> 태그의 다양한 속성

autofocus - 입력 커서 표시하기

페이지를 불러오자마자 폼의 요소 중에서 원하는 요소에 마우스 커서를 표시할 수 있음

placeholder - 힌트 표시하기

텍스트를 입력할 때 도움이 되도록 입력란에 힌트 내용 표시, 그 필드 클릭하면 내용이 사라짐

제목을 사용하지 않고도 어떤 내용을 입력해야하는지 알려줄 수 있어 편리

readonly - 읽기 전용 필드 만들기

사용자가 입력하지 못하고 읽게만

readonly="true" or readonly or readonly="readonly"

required - 필수 필드 지정하기

submit 버튼 클릭시 서버로 전송하는데, 이때 필수 필드에 필요한 내용이 채워졌는지 검사

필수적으로 입력해야 하는 내용에는 required 속성을 지정해 필수 필드로 만듬

required="required" or required

필수 필드는 브라우저에서 직접 체크하는 것이므로 오류 메시지 내용은 브라우저들마다 다르게 나타남

min, max, step 속성

해당 필드의 최솟값과 최댓값 지정

min : 최솟값

max : 최댓값

step : 허용된범위 내의 숫자의 간격

<input> 태그 유형이 date, datetime, datetime-local, month, week, time, number, range 일 경우에만 사용

size, minlength, maxlength 속성 - 길이, 최소 길이, 최대 길이 속성

size 속성은 한줄짜리 텍스트와 관련된 필드에서 몇 글자까지 화면에 보이게 할지 지정

maxlength 속성은 사용자가 최대 몇 글자까지 입력할 수 있는지 지정

minlength 속성은 사용자가 최소 몇 글자 이상을 입력해야하는지 지정

<select> <optgroup> <option> 태그 - 드롭다운 목록 만들기

사용자가 내용을 입력하는 것이 아니라, 여러 옵션 중에서 선택하도록 하고 싶을 때 드롭다운 목록을 사용

드롭다운 목록은 <select> 태그와 <option>태그를 이용해 표시

<select> 태그로 드롭다운 목록의 시작과 끝 표시

<option>태그를 이용해 항목을 추가. 여기서 value는 서버로 넘겨주기 위한 값을 지정

<textarea> 태그 - 여러 줄 입력하는 텍스트 영역 만들기

한 줄 이상의 문장을 입력할 때 사용하는 폼

게시물을 입력하거나 회원가입 양식에서 사용자 약관을 표시할 때 자주 사용

rows 세로 길이를 줄 단위로 지정 (지정한 숫자보다 줄 개수가 많아지면 스크롤 막대가 생김)

<button> 태그 - 버튼 넣기

<button> 태그의 type 속성은 버튼이 활성화 되었을 때 어떤 동작을 할지 지정

선택할 수 있는 값은 submit, reset, button 이고, 지정하지 않으면 submit으로 간주

속성 값 설명
submit 폼을 서버로 전송. <button type="submit">전송하기</button>와 같이 사용됨
reset 폼에 입력한 모든 내용을 초기화
button 버튼만 있을뿐 자체 기능은 없음
<input type="submit"> 이나 <input type="reset">, <input type="button"> 처럼 <input> 태그를 이용해 삽입 할 수도 있고,
<button>태그를 이용해 삽입할 수도 있슴

<button> 태그를 사용한 버튼에는 콘텐츠를 포함할 수 있기 때문에 아이콘을 추가할 수 있고, css 이용해 원하는 형태로 꾸밀 수도 있음

<progress> 태그 - 진행 상태 보여주기

작업 시작을 0으로 하고 최종 완료를 최댓값으로 해 얼마나 진척되었는지 숫자로 표현

이때 단위는 없고 표시하지 않음

<meter> 태그 - 값이 차지하는 크기 표시하기

진행 상황을 나타내는 <progress>태그

<meter>태그는 전체 크기 중에서 얼마나 차지하는지를 표현

<meter>태그는 진행된다는 의미보다 하드 디스크 전체 용량 중 현재 얼마나 사용하고 있는지를 나타내는 사용량이나 투표율처럼 지정된 범위 내에서 해당 값이 어느 정도 차지하는지 표현

<meter> 태그 속성
속성 설명
min, max 최솟값과 최댓값 (지정하지 않으면 0과 1로 간주)
value 범위 내에서 차지하는 값
low "이 정도면 낮다" 라고 할 정도의 값 지정
high "이 정도면 높다" 라고 할 정도의 값 지정
optimum "이 정도면 적당하다" 라고 할 정도의 범위 지정. optimum값이 high값보다 크다면 value값이 클수록 좋고, optimum값이 low값보다 작다면 value 값이 작을수록 좋음
