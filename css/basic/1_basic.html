웹에서 스타일은 미리 약속한 스타일 속성을 입력해 웹을 디자인하는 것

한 문서에서 반복해 쓰이는 스타일을 모아 놓은 것을 '스타일 시트'

내부 스타일 시트 - 웹 문서 안에서 사용할 스타일을 문서 안에 정리한 것을 '내부 스타일 시트'

스타일 정보는 <head>태그와 </head>태그 안에서 정의해야 하고, <style>태그와 </style> 태그 사이에 작성

외부 스타일 시트 - 사이트를 제작할 때 여러 웹 문서에서 사용할 스타일을 별도 파일로 저장해 놓고 필요할 때마다 가져와 사용하는 스타일 정보를 '외부 스타일 시트' (.css)

외부 스타일 시트를 연결할 때는 <style> 태그 없이 <link>태그만 사용해 미리 만들어 놓은 외부 스타일 시트 파일을 연결

인라인 스타일 - 스타일 시트를 사용하지 않고 스타일을 적용할 대상에 직접 표시

해당 태그에 style 속성을 사용해 style="속성: 속성 값;" 형태로 스타일을 적용

주요 선택자

스타일 속성을 적용하는 요소를 '선택자(selector)'

전체 선택자 - 모든 요소에 스타일 적용

  • 모든 요소에 적용할 때 사용
  • 전체 선택자로는 *(별표) 사용
  • 문서의 여백이나 글꼴 크기 등 기본 스타일을 초기화 할 때 사용

태그 선택자 - 특정 태그를 사용한 요소에 스타일 적용하기

  • 특정 태그가 쓰인 모든 요소에 스타일 적용
  • p 선택자를 정의하였다면 웹 문서의 모든 p 요소들에 스타일이 적용

span 선택자 - 텍스트 일부에만 클래스 스타일 적용

  • <span>태그를 사용해서 스타일을 적용할 텍스트를 묶고 클래스 스타일 지정

태그+클래스 선택자 - 특정 태그에만 클래스 선택자를 적용

  • accent인 클래스가 있을때 <h2> 태그에만 적용하려면 h2.accent 처럼 태그이름과 클래스 스타일 이름을 하나의 선택자로 사용

한꺼번에 둘 이상의 클래스 스타일 적용

  • 공백으로 구분해서 두 개 이상의 스타일을 나란히 적음

id 선택자 - 특정 부분에 스타일 적용

  • 클래스 선택자와 마찬가지로 웹 문서 안의 특정 부분에 스타일을 지정
  • 마침표(.) 대신 #기호를 앞에 사용
  • 클래스 선택자와의 차이는 클래스 선택자가 문서 안에서 여러번 사용할 수 있는 스타일인 반면, id 선택자는 문서 안에서 한번만 적용
  • id선택자는 중복해서 사용할 수 없음
  • 자바스크립트 프로그램에서 웹 요소들을 구별하기위해 자주 사용

group 선택자 - 둘 이상 요소에 같은 스타일 적용

  • 여러 선택자에 같은 스타일이 사용되는 경우 쉼표(,)로 구분해 여러 선택자를 나열한 후 스타일은 한번만 정의

css (캐스케이딩 스타일 시트)

  • '스타일 우선순위'는 캐스케이딩에서 가장 중요
  • 여기서 우선순위는 어떤 스타일을 먼저 적용할 것인지 결정하는 규칙
  1. 사용자 스타일 시트의 중요 스타일
    이 스타일 시트는 시스템을 통해 만들어진 것이기 떄문에 제작자가 제어할 수 없음
  2. 제작자 스타일 시트의 중요 스타일
    !important가 붙은 스타일
    제작자는 스타일 정의 시, 최우선으로 적용해야 할 스타일에는 !important 붙일 수 있음
  3. 제작자 스타일 시트의 일반 스타일
    지금까지 공부한 여러 스타일이 여기에 해당
  4. 기본적인 브라우저 스타일 시트
    브라우저들마다 기본적으로 지정하고 있는 스타일

적용 범위 (specificity)

  1. inline 스타일
    태그 안에 style 속성을 사용해 해당 태그에만 적용
  2. id 스타일
    지정한 부분에만 적용되는 스타일이지만, 한 문서 안에 한 번만 적용
  3. class 스타일
    지정한 부분에만 적용되는 스타일, 한 문서 안에 여러 번 적용 가능
  4. tag 스타일
    특정 태그에 똑같이 적용되는 스타일

스타일 상속

  • 자식 요소에서 별도로 스타일을 지정하지 않으면 부모 요소에 있는 스타일 속성들이 자식 요소로 전달되는데 이것을 '스타일 상속'이라 함
  • 하지만 모든 속성이 부모 요소에서 자식 요소로 상속되는 것은 아님
    부모 요소에 배경 이미지나 배경색이 있었다면 자식 요소에 상속되지 않음 (무한 반복되는 현상이 나타날 수 있기에)
    자식 요소의 배경 색이나 배경 이미지는 상속되지 않고 기본 값인 '투명' 으로 지정
