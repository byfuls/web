HTML5와 시맨틱 태그

HTML5 문서

시맨틱 태그가 사용된 HTML5 문서

  • 시맨틱 태그란 태그만 보고도 페이지 구조를 쉽게 이해할 수 있도록 정의된 태그
  • 사이트 제목이나 로고, 검색 창이 있는 헤더(header)
  • 여러 내용이 있는 본문(content)
  • 본문 외 내용을 나타내는 사이드바(sidebar)와 푸터(footer)

문서 구조를 위한 HTML5 시맨틱 태그

  • 사이트 별로 문서 구조는 약간씩 다르지만 보통 <header>태그와 <section>태그, <footer>태그 순으로 이루어짐

<header> 태그 - 머리말 지정하기

  • <header>태그는 특정 부분의 머리말에 해당
  • 헤더는 주로 페이지 맨 위쪽이나 왼쪾에 삽입하며
    헤더의 내용으로는 주로 <form>태그를 사용해 검색 창을 넣거나
    <nav>태그를 사용해 사이트 메뉴를 넣음

<nav> 태그 - 문서를 연결하는 내비게이션 링크

  • <nav>태그는 동일한 사이트 안의 문서나 다른 사이트의 문서로 연결하는 링크 모음
  • <nav>태그는 내비게이션 메뉴뿐만 아니라 푸터에 있는 사이트 링크 모음 부분에도 많이 사용

<section> 태그 - 주제별 콘텐츠 영역 나타내기

  • 콘텐츠가 들어가는 부분
  • <section>태그는 문서에서 콘텐츠 영역늘 나타냄
  • <section>태그는 콘텐츠를 주제별로 묶을 때 사용하며, 그 안에는 섹션 제목을 나타내는 <h1>~<h6> 제목 태그가 함께 사용
    (<section>태그 안에 또 다른 <section>태그를 넣을 수도 있음)
    (스타일을 적용하거나 스크립트를 사용하거나 단순히 영역을 나누기 위해 컨텐츠의 일부를 묶고 싶다면 <section>태그 대신 <div>태그를 사용하면 됨)

<article> 태그 - 콘텐츠 내용 넣기

  • 웹 상의 실제 내용을 넣음
    (보통 블로그의 포스트나 웹 사이트의 내용, 사용자가 등록한 코멘트, 독립적인 웹 콘텐츠 항목이 해당됨)
  • <section>태그와 <article>태그를 혼동하는데,
    <section>태그는 문맥 흐름 중에서 콘텐츠를 주제별로 묶을때 사용
    (<article>태그 안에 <section>태그를 넣을 수도 있음)

<aside> 태그 - 본문 이외의 내용 표시하기

  • 사이드바를 만드는 태그가 <aside>
  • 필수 요소는 아님 (보통 광고나 링크 모음 등 문서의 메인 내용에 영향을 미치지 않는 내용들을 넣을 때만 사용)
  • <aside>태그는 본문 내용 외에 주변에 표시되는 기타 내용들을 나타냄

<iframe> 태그 - 외부 문서 삽입하기

  • 웹 문서 안에 다른 웹 문서를 가져와 표시하는 것을 인라인 프레임(inline frame)
  • 인라인 프레임을 삽입하는 태그는 <iframe>

<footer> 태그 - 제작 정보와 저작권 정보 표시

  • 일반적으로 웹 문서 끝자락에 들어가는 <footer>태그 안에는 사이트 제작자의 연락처 정보와 저작권 정보를 표시
  • <footer>태그 안에는 <header>태그를 비롯해 <section>, <article>등 다른 레이아웃 태그들을 모두 사용할 수 있고 다양한 정보를 넣을 수 있음

<address> 태그 - 사이트 제작자 정보, 연락처 정보 나타내기

  • <address>태그는 주로 <footer>태그 안에 사용되는데 웹 페이지 제작자의 이름이나 제작자의 웹 페이지 또는 피드백을 위한 연락처 정보를 넣는데 사용

<div> 태그는 언제 사용할까?

  • HTML5에서는 주로 콘텐츠를 묶어 시각적 효과를 적용할 때 즉 콘텐츠에 CSS를 적용할 때 <div>태그를 사용
