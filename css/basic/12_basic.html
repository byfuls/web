미디어 쿼리

미디어 쿼리 이해하기

  • 웹 사이트에 접속할 수 있는 기기들이 늘면서 하나의 웹 사이트를 기기마다 다르게 표현하는 반응형 웹이 기본으로 되고 있음
  • 반응형 웹 디자인은 사이트에 접근하는 기기의 해상도에 따라 서로 다른 스타일 시트를 적용해 주는 미디어 쿼리를 기반으로 함

미디어 쿼리란?

  • 미디어 쿼리(Media Query)는 CSS3 모듈 중 하나로 사이트에 접속하는 장치에 따라 특정한 CSS 스타일을 사용하도록 해 줌
    접속하는 기기의 화면 크기에 따라 레이아웃이 달라짐
  • 사용자가 어떤 미디어를 사용하는가에 따라 사이트의 형태가 바뀌도록 CSS를 작성하는 방법을 미디어 쿼리
    (참조: https://mediaqueri.es/)

미디어 구문

  • 미디어 쿼리는 @media 속성을 사용해 특정 미디어에서 어떤 CSS를 적용할 것인지 지정해줌
  • 미디어 쿼리 구문은 <style>과 </style>사이에 사용하며 대,소문자를 구별하지 않음
  • 기본적으로 미디어 유형이 지정되어야 하고 필요할 경우 , and 연산자로 조건을 적용

미디어 유형의 종류

미디어 유형 사용 가능한 미디어
all 모든 미디어 유형
print 인쇄 장치
screen 컴퓨터 스크린(스마트폰 스크린 포함)
tv 음성과 영상이 동시 출력되는 TV
aural 음성 합성 장치(주로 화면을 읽어 소리로 출력해 주는 장치)
braille 점자 표시 장치
handheld 패드(pad)처럼 손에 들고 다니는 장치
projection 프로젝터
tty 디스플레이 기능이 제한된 장치(픽셀(px) 단위를 사용할 수 없음)
embossed 점자 프린터

미디어 쿼리의 조건

  • 미디어 쿼리는 특정 조건에 따라 적용할 CSS를 다르게 정의하므로 조건을 어떻게 체크할 것인지가 중요
  • 미디어 쿼리에서 사용하는 조건에는 주로 화면 크기와 관련된 것들이 많음

웹 문서의 가로 너비와 세로 높이

  • 실제 웹 문서의 내용이 화면에 보이는 영역을 '뷰포트'라 하는데,
    뷰포트의 너비와 높이를 미디어 쿼리의 조건으로 사용할 수 있음
  • 이때 height 값은 미디어에 따라 달라지기 때문에 주의해야함
  • 미디어 쿼리를 이용하면 예제와 같이 max-width값(브라우저 창의 최대 너비 값)에 따라 문서의 배경 이미지가 달라지도록 미디어 쿼리 구문을 구성 할 수 있음

단말기의 가로 너비와 세로 높이

  • 단말기에서도 기본적으로 제공하는 브라우저 창의 물리저 까로 너비와 세로 높이를 체크해 반응할 수 있는데
    주의할 점은 대부분의 단말기들은 단말기 해상도와 실제 브라우저의 너비가 다르다는 것
    예를 들어 아이폰4의 해상도는 640x960이지만 아이폰4의 사파리 브라우저의 너비는 320x480 임
  • 단말기 크기와 뷰포트 크기를 하나로 통일해 사용하기 위해 뷰포트를 지정할 때 width="device-width"로 놓고 사용함

화면 회전

  • 기기를 세로나 가로로 보는데 그때마다 웹 사이트의 화면 방향이 달라짐
  • 미디어 쿼리를 작성할 경우, orientation속성을 사용하면 화면 방향을 체크할 수 있음
  • orientation속성은 portrait 값과 landscape 값을 사용할 수 있음
    풍경화처럼 가로(width 값)가 넓으면 landscape이고,
    초상화처럼 세로(height 값)가 길면 portrait

미디어 쿼리 중단점 만들기

  • 미디어 쿼리를 작성할 때 서로 다른 CSS를 적용할 화면 크기를 중단점(break point)라고 함
  • 이 중단점을 어떻게 지정하는가에 따라 CSS가 달라지고 화면 레이아웃이 바뀌는데 대부분 기기의 화면 크기를 기준으로 함
  • 처리 속도나 화면 크기 등에서 다른 기기보다 모바일 기기의 제약 조건이 더 많기 때문에 모바일 기기의 레이아웃을 기본으로 해 CSS를 만듬
  • 출시된 기기들이 많고 앞으로도 다양한 크기의 기기들이 추가될 것이기 떄문에 어떻게 중단점을 나눌 것인지에 대한 표준은 없음
  • 참고로 CSS프레임워크인 부트스트랩(bootstrap)과 쇼핑몰 개발 플랫폼인 마젠토(Magento)에서는 다음과 같은 중단점을 사용

미디어 쿼리 적용하기

  • 미디어 쿼리는 웹 문서 안에서 @media 구문 다음에 조건에 맞는 CSS규칙을 직접 추가해 구현할 수 있고
  • 각 미디어 조건에 맞는 별도의 CSS파일을 만들어 <link>태그로 연결해 사용할 수도 있음

외부 CSS 파일 연결하기

  • 웹 문서에 적용하는 방법은 크게 외부 파일로 연결하는 방법과 직접 정의하는 방법이 있음

[외부 파일 적용] <link> 태그 사용하기

  • 외부 스타일 시트 파일 연결할 때 <link>태그를 이용하는 방법을 가장 많이 사용
  • <link> 태그는 <head>태그와 </head> 태그 사이에 삽입

[외부 파일 적용] @import 구문 사용하기

  • 외부 CSS파일 연결할 때 <link>태그 대신 @import 구문을 사용할 수 있는데
    @import 구문은 CSS 정의하는 <style> 태그와 </style> 태그 사이에서 사용

[외부 파일 적용] <link> vs @import 구문

  • 두 개 모두 외부 CSS파일을 가져와 사용하는 것이기 떄문에 CSS 파일이 1~2개 밖에 없다면 속도나 처리 면에서 큰 차이가 없다
  • 하지만 CSS 파일이 많다면, 이럴 경우 @import 구문보다 <link>태그가 안정적이고 빠르기 때문에 <link> 태그를 사용하길 권장함
  • 특히 인터넷 익스플로러 경우, @import구문과 자바스크립트가 있을 경우
    자바스크립트를 먼저 다운로드하고 @import구문에 있는 CSS를 다운로드 하기 때문에
    자바스크립트에서 스타일 관련 정보를 처리할 경우 오류를 낼 여지가 있음
  • 따라서 CSS파일이 많고 대규모 사이트를 개발할 때는 @import 구문보다 <link> 태그를 주로 사용

웹 문서에서 직접 정의하기

  • 웹 문서에서 직접 정의하는 방법은 크게 두가지
  • 첫 번째 방법은 <style> 태그 안에서 media속성을 사용해 조건을 지정하고 그 조건에 맞는 스타일을 정의하는 것
  • 두 번째 방법은 스타일을 선언할 때 @media 구문을 사용해 각 조건별로 스타일을 지정해 놓고 선택적으로 스타일을 적용하는 방법
  • 첫 번째 방법은 하나의 <style>태그 안에서 하나의 조건을 지정하지만
    두 번째 방법은 <style> 태그 안에 여러 조건에 따른 스타일을 모두 나열해 놓고 그 중 선택적으로 스타일을 사용

미디어 쿼리를 사용해 웹 문서 만들기

  • 브라우저 화면 너비에 따라서 레이아웃이 바뀌게 해볼 텐데 그렇게 하려면 CSS를 사용해 레이아웃 만드는 것에 익숙해져아함

레이아웃 구상하기

  • 반응형 웹 디자인은 화면 해상도에 따라 화면 요소들을 옮길 수 있어야 하므로 처음부터 레이아웃을 잘 구상하고 옮길 부분을 결정해야 함
  • 주로 모바일 화면부터 시작해 태블릿과 PC 화면 순서대로 구상하면 편함
    (모바일화면, 그림과 텍스트를 세로로 나열)
    (테블릿화면, 그림과 텍스트를 가로로 나열)
    (PC화면, 그림과 텍스트를 계속해 가로로 나열)

미디어 쿼리 중단점 결정하기

  • 미디어 쿼리를 사용하려면 중단점을 먼저 결정해야함
  • 화면 너비에 따라 카드를 가로로 배치하거나 세로로 배치하기 때문에 카드 너비 값만 정하면 미디어 쿼리 중단점을 정할 수 있음
  • 모바일 화면일 경우, 이미지와 텍스트를 세로로 배치하기 때문에 이미지 너비 값을 기준으로 카드 너비 값을 결정하면 됨
    사용할 이미지 너비가 300px이므로 카드의 너비도 300px로 지정하면 됨
  • 태블릿 화면일 경우, 이미지와 텍스트를 나란히 배치한 카드의 너비를 정하면 됨, 여기서는 550px로 정의하겠음
    이렇게 하면 왼쪽 이미지 너비는 300px가 되고 오른쪽 텍스트의 너비는 250px정도가 됨
  • 데스크톱 화면일 경우, 카드 3개를 나란히 배치
    카드 너비가 550px짜리 세 개와 카드마다 마진을 각각 10px로 지정할 생각이라면
    최소한 화면 너비가 1710px 이상이어야 함
  • 그렇다면 다음과 같이 미디어 쿼리 중단점을 지정
    768px 미만이면, 모바일 화면
    768px ~ 1719 사이이면, 태블릿 화면
    1720px 이상이면, 데스크톱 화면

태그로 구성하기

  • 지금까지 레이아웃 구상을 기본으로 HTML문서를 작성
  • 웹 문서의 다른 요소들에 영향을 주지 않으면서 위치를 바꾸려면 미디어 쿼리의 영향을 받는 요소를 <div>태그로 묶어줘야함
  • 여기서는 카드를 <div> 태그로 묶고 카드 안에 있는 텍스트 부분도 <div>태그로 묶음
