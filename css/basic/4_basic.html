css 박스 모델

css와 박스 모델

  • 웹 문서에서는 내용을 배치할 때 각 요소를 박스 형태로 구성, 'css 박스 모델'
    (웹 문서의 내용을 박스 형태로 정의하는 방법, 그 안에는 마진,패딩,테두리 등 여러 겹의 박스가 있음)

블록 레벨 요소와 인라인 레벨 요소

  • 박스 모델 => 블록 레벨 or 인라인 레벨
  • 블록 레벨
    • 태그를 사용해 요소를 삽입할 때 혼자 한 줄을 차지하는 요소 (해당 요소의 너비가 100% 라는 의미)
    • 따라서 그 요소의 왼쪽이나 오른쪽에 다른 요소가 올 수 없음
    • 너비나 마진, 패딩 등을 잉요해 크기나 위치를 지정하려면 블록 레벨 요소여야함
      <div>태그나 <p>태그 등이 블록 레벨 요소를 만드는 대표적인 태그
      (마진 = 요소들 사이의 여백, 패딩 = 테두리와 내용 사이의 여백)
  • 인라인 레벨
    • 줄을 차지하지 않는 요소
    • 즉, 화면에 표시되는 콘텐츠만큼만 영역을 차지하고 나머지 공간에는 다른 요소가 올 수 있음
    • 따라서 한줄에 여러개의 인라인 레벨 요소를 표시할 수 있음
      <img>태그나 <strong>태그 등이 인라인 요소를 만드는 태그

박스 모델(box model) - 박스 형태의 컨텐츠

  • 컨텐츠(content) = 내용을 차지하는 부분
  • 패딩(padding) = 박스와 컨텐츠 영역 사이의 여백
  • 테두리(border) = 박스의 테두리
  • 마진(margin) = 여러 박스 모델 사이의 여백

네 방향의 스타일을 따로 설정할 수 있음

width, height 속성 - 콘텐츠 영역의 크기

  • 박스 모델의 너비를 지정하는 width
  • 박스 모델의 높이를 지정하는 height

display 속성 - 화면 배치 방법 결정하기

  • display 속성을 이용하면 블록 레벨 요소를 인라인 레벨 요소로, 인라인 레벨 요소를 블록 레벨 요소로 바꿀 수 있음
  • 세로로 표시되는 목록을 가로 내비게이션으로 바꿀 때,
    한 줄로 표시되는 이미지에 여백과 테두리를 추가해 갤러리로 표시할 때 이 방법을 사용
  • 원래 display 속성은 해당 요소가 화면에 어떻게 보일지를 지정할 때 사용

테두리 관련 속성들

  • 테두리는 텍스트 단락뿐만 아니라 그림, 표, 양식 등 사각 형태를 가진 어떤 요소에나 적용 가능

border-style 속성 - 테두리 스타일 지정

  • 실선, 점선, 이중선 등 테두리의 스타일을 지정

border-width 속성 - 테두리 두께 지정

  • 테두리 두께 지정
  • border-width 속성으로 위,오른쪽,아래,왼쪽 순서로 한꺼번에 지정 가능
    속성 값이 2개라면, 위아래 + 좌우 묶어 지정
    속성 값이 4개라면, 위, 오른, 아래 왼쪽 으로 적용

border-color 속성 - 테두리 색상 지정

  • 박스 모델에서 테두리 색상을 지정
  • border-color 속성으로 위,오른쪽,아래,왼쪽 순서로 네 방향 테두리의 색상을 한꺼번에 지정 가능

border 속성 - 테두리 스타일 묶어 지정

  • 스타일, 두께, 색상 등 여러 속성을 묶어 표현
  • 네 방향의 테두리 스타일이 같다면 간단히 border 속성 이용
  • 두께와 색상, 스타일 순서는 상관없음

border-radius 속성 - 박스 모서리 둥글게

  • 박스 모서리 부분을 다양한 형태로 처리 가능

box-shadow 속성 - 선택한 요소에 그림자 효과 내기

  • 포토 앨범 등에서 사진 주변에 그림자를 추가하면 멋진 효과낼 수 있음
  • 그림자는 div 전체 등 지정하는 것에 따라 넣을 수 있음
  • box-shadow 속성 값에 따라 그림자의 위치를 바꿀 수 있고, 그림자의 색상이나 흐림 정도를 조절할 수 있음
  • box-shadow 속성에서 수평거리와 수직거리는 반드시 지정해야함
  • 그림자 효과는 포토 앨범뿐만 아니라 메뉴 등에 마우스를 올려놓으면 그 부분에만 그림자가 표시되도록 하고 싶을 때도 사용할 수 있음

여백을 조절하는 속성들

  • 박스 모델의 여백인 마진과 패딩
  • 마진과 패딩은 이미지나 텍스트 등 여러 요소를 자유롭게 배치할 때 유용

margin 속성 - 요소 주변 여백 설정하기

  • 마진은 현재 요소 주변의 여백
  • 따라서, 마진을 이용하면 한 요소와 다른 요소 사이의 간격을 조절 할 수 있음
  • margin-top, margin-right, margin-bottom, margin-left => margin
  • margin 속성 값에 하나만 있다면, 모든 방향에 적용
  • margin 속성 값에 두개만 있다면, 첫 번째 값은 마주보는 top, bottom 두 번째 값은 right, left 값
  • margin 속성 값에 세개만 있다면, 첫 번째 값은 top 두 번째 값은 right, left 세 번째 값은 bottom
  • margin-left 와 margin-right를 auto로 지정하면 요소의 너비 값을 뺀 나머지 공간의 좌 우 마진을 똑같이 맞춤
    (이 방법은 웹 요소를 중앙에 배치하려고 할 때 자주 사용)

margin overlap - 마진 중첩 현상

  • 박스 모델의 마진 속성을 사용할 때 주의
    특히 세로로 배치할 경우, 마진과 마진이 만날 때 마진 값이 큰 쪽으로 겹쳐지는 것 => 마진 중첩(margin overlap), 마진 상쇄(margin collapse)
  • 여러 요소를 세로로 배치할 때 맨 위의 마진과 맨 아래 마진에 비해 중간에 있는 마진들이 너무 커지는 것을 방지하기 위한 현상
  • 즉, 마진 중첩은 아래 마진과 위 마진이 서로 만날 때 큰 마진 값으로 합쳐지는 것
  • 오른쪽 마진과 왼쪽 마진이 만날 경우에는 중첩되지 않음

padding 속성 - 콘텐츠 영역과 테두리 사이 여백 설정하기

  • 패딩이란, 컨텐츠 영역과 테두리 사이의 여백
  • 즉, 테두리 안쪽의 여백
  • padding-top, padding-right, padding-bottom, padding-left => padding
    (margin 속성과 사용법이 비슷)
